## 專案概述

**astro-bot** 是台灣天文攝影專用 LINE Bot，結合天文計算、氣象預報與 AI 自然語言回覆。

- **Repo**：`dannytsao/astro-bot`
- **部署**：Render（Web Service，Flask Webhook，24 小時持續運行）
- **語言**：Python 3
- **主要檔案**：`main.py`（單檔架構）
- **備份**：`main_telegram.py`（舊版 Telegram 實作，保留備用）

---

## 技術架構

```
用戶 LINE 訊息（Webhook → Flask）
    ↓
意圖解析（Claude API - parse_intent）
    ↓
超出範圍攔截（check_unsupported）
    ↓
天文計算（Skyfield + de421.bsp）
  - 月出月落、天文薄暮
  - 有效暗空窗口
  - 銀河核心構圖方位角
    ↓
氣象預報（Open-Meteo API）
    ↓
氣象狀態評估（weather_status）
    ↓
回覆生成（Claude API - generate_reply，動態 system prompt）
    ↓
反饋記錄（Google Sheets API）
```

## 目前版本：Phase 2（已完成）

### 新增功能
- 月出月落時刻與方位角計算
- 天文薄暮（太陽 < -18°）計算
- 有效暗空窗口（剔除月光時段）
- 銀河核心構圖方位角＋月亮干擾評估
- 回覆新增【月亮窗口】和【銀河構圖方位】區塊
- 氣象優先邏輯（直接用雲量判斷 weather_status）
- 動態 system prompt（依氣象狀態調整回覆深度）
- 超出範圍攔截：行星、日食月食、未知彗星（IAU 命名格式）
- 超出15天預報範圍提示

### 氣象狀態四種情境
| 狀態 | 條件 | LLM 行為 |
|---|---|---|
| `out_of_range` | 超出15天預報 | 結論只談天文，明確標註氣象未知 |
| `bad` | 雲量 > 80% | 結論直接說不適合，其他區塊簡化 |
| `unstable` | 雲量 40~80% | 結論標註不穩定，建議當天再確認 |
| `good` | 雲量 ≤ 40% | 正常完整分析 |

## 第三階段計劃（進行中）— 氣象精準化 + 決策強化

> 核心方向：從「單純雲量判斷」升級為「多維度微氣候交叉分析」，讓攝影師直接拿到 Go/No-Go 結論，不需要自己判讀數據。

### 3A：決策引擎強化（改 prompt，零新 API）
| 優先 | 項目 | 說明 |
|---|---|---|
| 🔴 最高 | 信心度指標（CCI） | 目前只有 ✅/⚠️/❌，加入「信心度 XX%」量化不確定性 |
| 🔴 最高 | 紅藍軍對抗邏輯 | 回覆加入風險反方觀點，避免 LLM 過度樂觀 |
| 🔴 高 | 結露臨界值收緊 | 從 3°C 改為 1.5°C（T−Td），更符合實際起霧門檻 |
| 🔴 高 | 動態格式 | 銀河構圖區塊只在查詢銀河時出現，減少雜訊 |

### 3B：氣象資料升級（需新 API）
| 優先 | 項目 | 說明 |
|---|---|---|
| 🔴 最高 | Clear Outside API | 雲層分層（低/中/高雲），天文攝影專用預報 |
| 🔴 高 | 垂直濕度梯度（RH Gradient） | 判斷水氣是形成「雲海」還是「白牆（濃霧）」，解決低海拔地形霧陷阱 |
| 🟡 中 | 大氣透明度（Transparency） | 霾和水氣影響星點清晰度，7Timer 已有部分資料 |
| 🟡 中 | 視寧度 Seeing Index | Meteoblue，影響星點銳利度 |
| 🟡 中 | 微型氣候修正 | 各地點歷史氣候特徵（如合歡山午後雷陣雨） |
| 🟢 低 | 光害指數 | 靜態資料，各地點背景天空亮度 |

### 3C：器材與拍攝建議（改 prompt）
| 優先 | 項目 | 說明 |
|---|---|---|
| 🔴 高 | 基礎曝光參數建議 | 根據月相、天體類型給出 500 法則快門、建議 ISO、光圈 |
| 🟡 中 | 除霧加熱帶提醒 | 結露風險觸發時主動提醒開啟，並說明原因 |
| 🟡 中 | 智慧望遠鏡警告（Seestar/DWARF） | 低仰角大氣折射、風速追蹤微震、光害濾鏡建議 |

### 3D：其他功能
| 優先 | 項目 | 說明 |
|---|---|---|
| 🔴 高 | 彗星即時座標 | JPL Horizons API，取代固定近似值 |
| 🟡 中 | 備案地點建議（Elevation Pivot） | 低地起霧時自動推薦附近破霧高度以上的替代地點 |
| 🟡 中 | 回覆速度優化 | 目前 30~60 秒，兩次 API call 是瓶頸 |
| 🟢 低 | visibility 欄位整合 | Open-Meteo 已拉取但未使用 |

| 目標天體偏好 | 設定訂閱標的，只在特定天體條件符合時通知 |

### 主動推播
| 項目 | 說明 |
|---|---|

| 訂閱日期區間 | 設定觀測計劃期間，期間內自動監測並推播 |
| 推播時間設定 | 用戶自訂每日通知時間 |

### 閉環學習系統
| 項目 | 說明 |
|---|---|

## 工作原則

1. **天文計算準確性不可妥協**：仰角、方位角、月出月落時刻由 Skyfield 計算，LLM 只負責詮釋
2. **氣象優先**：氣象條件是第一道篩選，壞天氣直接結論，不浪費用戶時間看天文分析
3. **攔截順序**：`parse_intent` → `check_unsupported` → `run_query`，不支援的查詢不進入計算
4. **`match_targets` 找不到時回傳空陣列**，不 fallback 整個標的庫（避免卡死）
5. **批次修 Bug**：收集完所有問題再一起修，避免反覆 deploy
