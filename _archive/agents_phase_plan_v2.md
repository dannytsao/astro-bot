## 第四階段計劃（舊版：個人化訂閱）

> 注意：新版 ROADMAP 的 Phase 4 已重新定義為「出勤規劃層（How to Go）」。
> 本節的「個人化」內容已分配至新版 Phase 4B（個人化）與 Phase 5（訂閱與閉環學習）。

### 用戶個人化

| 項目 | 說明 |
|---|---|
| 用戶地點記憶 | 設定多個常用地點（如塔塔加、日月潭），不需每次輸入 |
| 攝影設備檔案 | 儲存焦段、相機型號、有無赤道儀，Bot 提供設備對應建議 |
| 目標天體偏好 | 設定訂閱標的，只在特定天體條件符合時通知 |
| 觀測計畫保存 | 使用者可儲存「6/20 合歡山銀河」，之後用「更新我的計畫」重新檢查條件 |

### 主動推播

| 項目 | 說明 |
|---|---|
| 每日自動推播 | 天況好時主動通知訂閱用戶 |
| 訂閱日期區間 | 設定觀測計劃期間，期間內自動監測並推播 |
| 推播時間設定 | 用戶自訂每日通知時間 |

### 閉環學習系統

| 項目 | 說明 |
|---|---|
| 預報誤差記錄 | 收集用戶回饋（👍👎），建立各地點預報誤差修正模型 |
| 黑天鵝標記 | 用戶可標記「意外大景」或「白跑一趟」，轉化為地點加權參數 |
| 診斷模式 | 預報與實際嚴重不符時啟動，記錄微氣候誤差因子 |

### 技術需求
- 用戶資料持久化（Redis 或 Google Sheets 擴充）
- 排程推播（Render Cron Job 或 APScheduler）
- 多用戶獨立設定管理

| 項目 | 說明 |

| CCTV 即時影像驗證 | Deferred：影像品質、來源穩定性、夜間可視性與誤判風險高，等核心決策引擎成熟後再評估 |
