## Phase 3A：出勤決策引擎（最高優先）

| 優先 | 功能 | 說明 |
| --- | --- | --- |
| 🔴 最高 | 信心度指標（CCI） | 在 ✅/⚠️/❌ 之外加入「信心度 XX%」，量化預報不確定性 |
| 🔴 最高 | 紅藍軍對抗邏輯 | 回覆加入風險反方觀點，避免 LLM 過度樂觀 |
| 🔴 最高 | 地點比較模式 | 支援「合歡山 vs 阿里山」這類查詢，直接比較哪裡更值得出勤 |
| 🔴 高 | 今晚 / 週末最佳地點 | 從 approved 地點中排名，回答「今晚哪裡最好拍」 |
| 🔴 高 | 多題材決策框架 | CCI 不只服務銀河，需可套用於星雲、流星雨、月景、日月行星運行景象與雲海 / 霧題材 |
| ~~🔴 高~~ | ~~結露臨界值收緊~~ | ~~T−Td 從 3°C 改為 1.5°C，更符合實際起霧門檻~~ ✅ 2026-06-10 |

## Phase 3B：規劃 UX 與資料營運

| 優先 | 功能 | 說明 |
| --- | --- | --- |
| 🔴 高 | 地點許願池處理流程 | 定期審核「地點許願（自動）」與使用者建議，補來源、座標、別名、可及性與安全注意事項後升級進地點資料庫 |
| 🟡 中 | 備案地點建議 | 低地起霧時推薦附近破霧高度以上的替代地點 |
| 🟡 中 | 雲海 / 霧判斷模式 | 針對「想拍雲海」「會不會起霧」建立獨立判斷語氣，先用濕度、露點差、海拔與地點特性，再視資料來源升級 |
| 🟡 中 | 日月行星景象規劃 | 支援月出月落、月相、日出日落方位、行星可見性、行星合月等攝影題材；未有可靠資料前不得讓 LLM 猜測 |
| 🟡 中 | 15 天日曆摘要 | 將 15 天景點氣象評估整理成每日 Go/No-Go 排名與 Top 3 詳情 |
| 🟡 中 | 回覆速度優化 | 目前 30~60 秒，兩次 LLM call 是瓶頸 |
| ~~🔴 高~~ | ~~曝光參數建議~~ | ~~500 法則快門、建議 ISO、光圈，依天體類型給出~~ ✅ 2026-06-10 |

## Phase 4：個人化與訂閱（舊定義，已由出勤規劃層取代）

| 優先 | 功能 | 說明 |
| --- | --- | --- |
| 🔴 高 | 用戶地點 / 設備記憶 | 儲存常用地點、鏡頭焦段、相機型號，提供個人化建議 |
| 🔴 高 | 觀測計畫保存 | 使用者可儲存「6/20 合歡山銀河」並日後更新條件 |
| 🟡 中 | 訂閱推播 | 設定觀測計劃期間，自動監測並推播 |
| 🟡 中 | 閉環學習 | 收集預報誤差，建立各地點修正模型；支援「黑天鵝」標記 |
